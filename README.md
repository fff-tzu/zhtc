
# 数据驱动的短时停车需求预测与动态引导系统
本仓库是一套完整的**数据驱动的短时停车需求预测与动态引导系统**工程，涵盖前端、后端、数据库初始化全链路代码与配置，旨在实现停车需求的智能预测、停车资源的动态引导与管理。

---

## 一、项目概述
系统聚焦城市短时停车需求场景，基于数据驱动的算法模型实现停车需求预测，并通过前端可视化、后端业务逻辑支撑，完成停车资源的动态引导与用户管理，核心解决**停车难、资源分配不均**问题,适配不同业务场景/部署环境，具备完整的用户身份管理、停车需求预测、资源引导调度能力。

---

## 二、技术栈
### 后端
- 开发语言：Java
- 构建工具：Maven（内置 `mvnw`/`mvnw.cmd` 跨平台脚本）
- 核心能力：业务逻辑、停车预测算法、数据库交互、接口服务

### 前端
- 包管理：npm
- 核心模块：登录认证、用户管理、停车数据可视化、动态引导展示

### 数据层
- 数据库：MySQL 5.7+/8.0+
- 初始化脚本：SQL 表结构 + 基础数据

## 五、安装部署指南
### 适用范围
### 步骤 1：数据库初始化
1. 启动 MySQL 服务
2. 创建数据库
```sql
CREATE DATABASE parking_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```
3. 执行初始化脚本
```bash
mysql -u root -p 你的密码 parking_system < sql/init_db.sql
```

### 步骤 2：后端启动（zhtc-master）
1. 修改数据库配置文件 `resources/application.yml`，配置账号密码
2. 启动服务
```bash
# Linux/macOS
./mvnw spring-boot:run
# Windows
mvnw.cmd spring-boot:run
```
3. 启动成功标志：`Started ZhtcApplication`

### 步骤 3：前端启动（zhtc-front）
1. 安装依赖
```bash
cd zhtc-front
npm install
```
2. 配置后端接口地址（可选）
3. 启动前端服务
```bash
npm run serve
```
4. 访问地址：`http://localhost:8081`

### 步骤 4：系统验证
1. 打开浏览器访问前端地址
2. 使用默认账号（`admin/123456`）登录
3. 正常使用停车预测、引导功能即部署成功

## 七、常见问题排查
| 问题现象 | 解决方案 |
|----------|----------|
| 后端数据库连接失败 | 检查 MySQL 服务、配置文件账号密码、数据库是否创建 |
| 前端无法访问后端 | 确认后端启动、接口地址配置正确、关闭防火墙 |
| npm 安装依赖失败 | 清理缓存、切换淘宝镜像、检查 Node 版本 |
| 端口冲突 | 修改后端 `server.port`、前端 `devServer.port` |

---

## 八、维护说明
1. **前端依赖更新**：`npm update` → 提交 `package-lock.json`
2. **后端依赖更新**：修改 `pom.xml` → 执行 `mvn clean install`
3. **数据库更新**：修改 `init_db.sql` → 备份数据后重新执行
4. **生产部署**：后端打包运行、前端打包部署 Nginx、数据库定时备份

---

## 九、补充说明
1. 前端依赖必须使用 `package-lock.json` 保证版本一致
2. 后端启动前必须确保数据库服务正常运行
3. 系统支持 Windows/Linux/macOS 全平台运行
